# 代码侦查

## 工作流程

0. 所有调用的输入和输出中涉及到路径的一律用绝对路径！！！
1. **search_file**：对目标目录执行工具得到 paths。paths 中没有的即项目不存在。特别重要，只对目标目录及其子目录进行侦查，不允许看其他目录。
2. 从 paths 中选关键文件读取，path 从 paths 原样复制。
3. **循环**：读一批代码文件 → 读入当前交付件 → 综合分析 → 更新交付件写入 → 读下一批 → 更新交付件写入。完成交付件结束。
4. 任务指定了报告路径时，必须 write_file 落盘。
